## How to Use — Incremental Implementation

Build one milestone at a time, feature by feature:

1. Copy the `product-plan/` folder to your implementation codebase
2. Start with **01-shell** (all sections depend on it)
3. For each milestone:
   - Open `prompts/section-prompt.md`
   - Replace the placeholder variables (`SECTION_NAME`, `SECTION_ID`, `NN`)
   - Copy the prompt into your coding agent
   - Answer the agent's clarifying questions about tech stack, auth, etc.
   - Review and test before moving to the next milestone
4. Repeat for milestones 02 through 07 in order

## Testing

Each section in `sections/[section-id]/` includes a `tests.md` file with UI behavior specs. These describe the expected interactions, role-based behavior, and edge cases for that section. Use them to verify your implementation or generate automated tests.

## Tips

- **Start with the Shell.** All sections depend on it for layout, navigation, i18n, and theme switching.
- **Use the data shapes.** The TypeScript interfaces in `data-shapes/overview.ts` define exactly what data each component expects. Wire them to your API layer or mock data.
- **Follow the design tokens.** The color and font specs in `design-system/` ensure visual consistency. Use Tailwind utility classes -- no custom CSS needed.
- **Props-based components.** All reference components accept data and callbacks via props. This makes them portable to any state management approach.
- **Role-based views.** Admin, teacher, and student see different UI. Check `product-overview.md` for sidebar visibility and section-level access rules.
- **i18n from day one.** Use react-i18next with namespace-per-section JSON files. Adding a language later is just adding a folder.
