Introducción

Este documento técnico describe la arquitectura, el stack tecnológico, el modelo de datos y las integraciones externas del Sistema Escolar CEICAVS. Fue elaborado en mi rol de Team Lead y desarrollador principal del proyecto, con el objetivo de documentar las decisiones de diseño tomadas durante el desarrollo.

El sistema se construyó como un monorepo con separación clara entre frontend (React SPA), backend (NestJS API GraphQL) y base de datos (PostgreSQL con Prisma ORM), siguiendo principios de arquitectura limpia y patrones CQRS para garantizar mantenibilidad y escalabilidad.

4. Flujo de Autenticación

1
El usuario envía email y password a la mutación GraphQL login.
2
El servidor verifica la contraseña con bcrypt.compare(). Si es correcta, genera accessToken (exp: 15 min) y refreshToken (exp: 7 días) firmados con JWT.
3
El frontend almacena ambos tokens en localStorage y configura el Apollo Link para adjuntar Authorization: Bearer <accessToken> en cada petición.
4
AppBootstrap rehidrata el contexto de autenticación al recargar la página leyendo el token de localStorage.
5
AbilityProvider llama a defineAbilityFor(role) de @ceicavs/shared y publica el objeto CASL en el contexto React.
6
Cuando el accessToken expira, el Apollo Link de refresco envía automáticamente la mutación refreshToken y retoma la petición original con el nuevo token.

Fuente única de verdad
packages/shared/src/casl.ts define los enums UserRole, Action, Subject y la función defineAbilityFor(role) usados tanto en el backend como en el frontend.
