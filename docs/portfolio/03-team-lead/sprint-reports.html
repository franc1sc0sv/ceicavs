Objetivo del Sprint

Implementar el sistema de transcripción de audio con inteligencia artificial, incluyendo el procesamiento en el navegador con Whisper ONNX y la generación de resúmenes con LLM (Groq/Gemini). También completar la importación masiva CSV postergada del sprint anterior.

Tareas Completadas

  • Importación masiva de usuarios desde CSV (arrastrada de S2)
  • Integración de Transformers.js con modelo Whisper ONNX en Web Worker
  • UI de transcripción: carga de audio, progreso en tiempo real
  • API backend para almacenamiento y actualización de transcripciones
  • Integración Groq SDK como proveedor primario de resumen
  • Integración Google Gemini como proveedor fallback
  • FallbackAIProvider con manejo de rate limit (HTTP 429)
  • Optimización: fragmentación de transcripciones largas (>15 KB)

Decisiones Técnicas

  • Se eligió @huggingface/transformers v4 sobre la versión anterior @xenova/transformers por mejor soporte WebGPU.
  • El modelo principal es onnx-community/whisper-small (WebGPU); fallback a whisper-base (WASM) para dispositivos sin WebGPU.
  • Groq se eligió como proveedor primario por su latencia menor frente a Gemini.

Blockers Encontrados

  • La primera carga del modelo Whisper tarda ~30 segundos en conexiones lentas (caché del modelo en el navegador resuelve las cargas posteriores).
