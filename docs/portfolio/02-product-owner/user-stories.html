Detalle de Historias — Sprint 1

Autenticación

US-001 Iniciar sesión con correo institucional
Must Sprint 1

Como usuario del sistema (administrador, docente o estudiante), quiero iniciar sesión con mi correo institucional y contraseña, para acceder a las funcionalidades correspondientes a mi rol.

Criterios de Aceptación
  • El formulario de login acepta correo electrónico y contraseña.
  • Si las credenciales son correctas, el sistema emite un access token y un refresh token JWT.
  • Si las credenciales son incorrectas, se muestra un mensaje de error sin revelar cuál campo es incorrecto.
  • El usuario es redirigido al dashboard apropiado según su rol.
  • El token es almacenado en localStorage y adjuntado a cada petición GraphQL.
US-002 Control de acceso por rol
Must Sprint 1

Como administrador del sistema, quiero que cada usuario solo pueda acceder a las funcionalidades permitidas para su rol, para garantizar la seguridad y privacidad de la información.

Criterios de Aceptación
  • El backend verifica el rol en cada operación mediante el guard CASL.
  • El frontend oculta rutas y controles no permitidos para el rol activo.
  • Un estudiante que intenta acceder a /people es redirigido al dashboard.
  • Las operaciones de escritura en el backend devuelven 403 si el usuario no tiene permiso.

Dashboard

US-003 Panel de administrador con estadísticas globales
Must Sprint 1

Como administrador, quiero ver un panel con estadísticas globales de la plataforma al iniciar sesión, para tener visibilidad del estado general del sistema.

Criterios de Aceptación
  • El panel muestra el conteo de usuarios por rol (Admin, Docente, Estudiante).
  • Muestra el número de publicaciones del blog por estado (publicadas, en revisión).
  • Muestra estadísticas generales de asistencia y actividad reciente.
  • Incluye accesos rápidos a los módulos principales.
US-004 Panel de docente con estadísticas de grupos
Must Sprint 1

Como docente, quiero ver un panel con estadísticas de mis grupos al iniciar sesión, para hacer seguimiento rápido de la asistencia y actividad de mis estudiantes.

Criterios de Aceptación
  • El panel muestra solo los grupos asignados al docente.
  • Muestra el porcentaje de asistencia reciente por grupo.
  • Incluye accesos rápidos para tomar asistencia.

Detalle de Historias — Sprint 1 (continuación)

Gestión de Personas

US-006 Crear, editar y eliminar usuarios
Must Sprint 1

Como administrador, quiero crear, editar y eliminar cuentas de usuario desde el panel de personas, para mantener actualizado el directorio del Centro Escolar.

Criterios de Aceptación
  • El formulario de creación solicita nombre, correo, contraseña y rol.
  • El formulario valida que el correo sea único antes de guardar.
  • La edición permite cambiar nombre, correo, contraseña y rol.
  • La eliminación es lógica (soft delete) y reversible por el administrador.
  • Se puede seleccionar múltiples usuarios para eliminar en masa.
US-007 Importar usuarios desde CSV
Should Sprint 1

Como administrador, quiero cargar un archivo CSV con información de usuarios, para crear múltiples cuentas de forma masiva sin ingresar cada una manualmente.

Criterios de Aceptación
  • El sistema acepta archivos CSV con columnas: nombre, correo, contraseña, rol.
  • Se muestra una vista previa de los registros antes de confirmar la importación.
  • Los registros duplicados (mismo correo) son ignorados con advertencia.
  • Se muestra un resumen de registros creados y omitidos al finalizar.

Sistema de Asistencia

US-009 Registrar asistencia de un grupo
Must Sprint 1

Como docente, quiero marcar el estado de asistencia de cada estudiante en mi grupo, para llevar un registro preciso de la presencia en clase.

Criterios de Aceptación
  • El roster muestra todos los estudiantes del grupo con su foto o iniciales.
  • Los estados disponibles son: Presente, Ausente, Tardanza, Justificado.
  • El registro se guarda con la fecha y hora del servidor, no del cliente.
  • No es posible registrar asistencia para una fecha futura.
  • Si ya existe un registro para la fecha, se permite editar.
US-010 Ver reporte de asistencia por período
Must Sprint 1

Como docente o administrador, quiero ver un reporte de asistencia del grupo para un período seleccionado, para identificar patrones de inasistencia y tomar decisiones.

Criterios de Aceptación
  • El reporte ofrece filtros por período: semanal, mensual y personalizado.
  • La tabla muestra por cada estudiante: total de presencias, ausencias, tardanzas y justificados.
  • El estudiante solo puede ver su propia fila en el reporte.
  • Los datos se actualizan al cambiar el filtro de período sin recargar la página.

Detalle de Historias — Sprint 2

Blog Institucional

US-014 Crear publicaciones de blog
Must Sprint 2

Como docente o administrador, quiero redactar y publicar artículos en el blog institucional, para compartir información relevante con la comunidad educativa.

Criterios de Aceptación
  • El formulario incluye campos: título, extracto, contenido rico, categoría e imágenes.
  • El docente y admin pueden publicar directamente (sin revisión).
  • El estudiante solo puede enviar para revisión; el sistema crea un borrador pendiente.
  • Las imágenes adjuntas se muestran en el cuerpo de la publicación.
US-016 Revisar publicaciones de estudiantes
Should Sprint 2

Como docente o administrador, quiero revisar, aprobar o rechazar las publicaciones enviadas por estudiantes, para mantener la calidad del contenido del blog.

Criterios de Aceptación
  • La cola de revisión muestra todas las publicaciones en estado «pendiente».
  • El revisor puede leer el contenido completo antes de tomar una decisión.
  • Aprobar una publicación la pone en estado «publicada» y la hace visible en el feed.
  • Rechazar requiere una nota explicativa que se muestra al estudiante.

Herramientas Educativas

US-019 Acceder al catálogo de herramientas
Must Sprint 2

Como usuario de la plataforma, quiero explorar el catálogo de herramientas educativas organizadas por categoría, para encontrar rápidamente la utilidad que necesito en clase.

Criterios de Aceptación
  • El catálogo muestra las 14 herramientas disponibles en una cuadrícula.
  • Cada herramienta tiene nombre, descripción breve e ícono de categoría.
  • El usuario puede marcar herramientas como favoritas para acceso rápido.
  • Las herramientas se abren sin necesidad de instalar software adicional.

Detalle de Historias — Sprint 3

Sistema de Transcripción con IA

US-024 Subir audio para transcribir automáticamente
Must Sprint 3

Como docente o administrador, quiero subir un archivo de audio de una reunión o clase, para obtener automáticamente la transcripción del contenido hablado.

Criterios de Aceptación
  • El sistema acepta formatos MP3, WAV y M4A.
  • La transcripción se procesa completamente en el navegador usando Whisper ONNX (sin enviar el audio a servidores externos).
  • El progreso de la transcripción se muestra en tiempo real con barra de avance.
  • El resultado se guarda en la plataforma asociado al usuario.
  • En caso de error, se muestra un mensaje descriptivo con opción de reintentar.
US-026 Generar resumen automático con puntos clave
Must Sprint 3

Como docente, quiero generar un resumen estructurado de la transcripción con puntos clave y elementos de acción, para aprovechar el contenido de la reunión sin leer el texto completo.

Criterios de Aceptación
  • El resumen se genera mediante el proveedor de IA configurado (Groq / Gemini).
  • El resultado incluye: resumen ejecutivo, lista de puntos clave y lista de elementos de acción.
  • La generación tarda menos de 30 segundos para transcripciones menores a 10,000 palabras.
  • El resumen se muestra en una pestaña separada sin reemplazar la transcripción original.
  • Si el proveedor primario falla por límite de tasa, el sistema usa automáticamente el proveedor alternativo.
US-027 Editar la transcripción generada
Should Sprint 3

Como docente, quiero editar manualmente el texto de una transcripción para corregir errores de reconocimiento, para mejorar la precisión del documento antes de compartirlo.

Criterios de Aceptación
  • El campo de transcripción es editable directamente en pantalla.
  • Los cambios se guardan explícitamente al hacer clic en «Guardar».
  • Si el usuario navega sin guardar, se muestra una advertencia de confirmación.
